核心能力

复杂写作需要的，不是更长的聊天记录

TreeTalk 管理的是写作背后的思考结构：一个问题可以生长出多条路线，任何路线都能返回，过去的背景随时可以重新进入当前创作。

⑂
01 / 分层多分支

主线、人物与伏笔，各自深入但不彼此打断

从章节根节点建立多条创作分支。每个子问题继承父层背景，写完支线后可以立即返回原来的主线。

⌁
02 / 记忆库

人物卡、世界观和前文伏笔，随时引用

在当前分支搜索背景资料，将选中的记忆带入 AI 上下文，减少设定遗忘与前后冲突。

▱
03 / 标签页

不同创作任务分开进行

章节推演、角色对话、世界观补充可以放在不同标签页，切换时保留各自状态。

⌖
04 / 精准追问

框选一句话，直接从它继续生长

无需重新复制整段背景。选择回答中的文字、代码或公式，就能创建带完整父级上下文的新分支。

小说创作示例

从一章的四条思路，写成一张不会丢失的剧情地图

你可以同时探索剧情推进、人物动机、隐藏伏笔与世界规则。TreeTalk 不替你决定故事，而是让所有可能性都有清晰的位置。

  1. 建立章节根节点写清本章目标与已有背景。
  2. 把问题拆成不同分支主线、人物、伏笔和设定分别深入。
  3. 从记忆库补充背景调用人物卡、时间线和前文线索。
  4. 回到父层整合选择保留探索过程，再决定最终写法。

常见问题

下载之前，你可能想知道

TreeTalk 会把我的作品上传到自己的服务器吗？

TreeTalk 采用本地优先设计，标签页、问题结构、知识与配置主要保存在你的设备中。使用模型时，当前问题和你主动引用的上下文会发送给你选择的 AI 服务商或 ChatGPT 网页服务。

一定需要 API Key 吗？

不一定。你可以使用 API 模式连接兼容的模型服务，也可以使用浏览器桥接模式登录自己的 ChatGPT 网页账户。

TreeTalk 是笔记软件吗？

它更像结构化 AI 对话与创作工作台，侧重获取、展开和回溯思路。正式排版和长期笔记仍可以交给你熟悉的写作或笔记工具。

现在支持哪些系统？

当前公开构建面向 Windows x64、macOS Apple Silicon 和 macOS Intel。发布页会同时提供安装包、便携包或校验文件。
