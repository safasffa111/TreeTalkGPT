## TreeTalk 是什么

TreeTalk 是一个面向学习、资料理解、代码分析和复杂问题探索的 AI 桌面客户端。

普通 AI 聊天工具通常把所有问题和回答放在一条不断变长的时间线上。随着追问次数增加，用户很容易遇到这些问题：

- 想深入研究回答中的一个知识点，却会打断原来的问题；
- 追问很多层以后，很难返回最初的问题；
- 多个研究方向混在同一个聊天记录中；
- 想针对一行代码、一个公式或一句话提问时，需要反复复制上下文；
- 聊天结束后只剩一条很长的记录，难以看出自己的思考路径；
- 有价值的问答无法方便地保存为长期知识材料。

TreeTalk 使用**问题树、层级追问和本地知识仓库**重新组织这个过程。

```text
传统聊天：问题 → 回答 → 问题 → 回答 → 问题 → 回答……

TreeTalk：
主问题
├── 子问题 A
│   ├── 深入问题 A-1
│   └── 深入问题 A-2
├── 子问题 B
└── 子问题 C
```

用户可以从任意回答中选择一个知识点继续追问，完成后返回父问题，再进入另一个分支。最终保存的不只是一个答案，而是完整的知识探索路径。

## 产品定位

TreeTalk 的目标不是替代 Obsidian、Notion、Logseq 等专业笔记软件。

它更关注 AI 获取知识的前半段：

1. 提出问题；
2. 阅读回答；
3. 发现不理解或值得继续研究的内容；
4. 针对局部内容继续追问；
5. 在不同问题分支之间切换；
6. 将有价值的问题结构保存下来；
7. 再根据需要整理到其他笔记软件中。

因此，TreeTalk 更像是一个**结构化 AI 对话工作台**，而不是传统意义上的笔记编辑器。

## TreeTalk 解决了什么需求

### 1. 解决 AI 对话只能线性发展的限制

一个回答往往包含多个知识点。TreeTalk 会为不同研究方向建立独立的问题分支，让每个方向都有自己的上下文，避免所有追问混在一条时间线里。

### 2. 解决深入追问后无法快速返回的问题

TreeTalk 会记录当前问题与父问题之间的关系。用户可以连续深入多个层级，随后返回上一级，或者通过问题树直接跳转到任意历史节点。

### 3. 解决局部内容追问不准确的问题

TreeTalk 支持框选文字、公式或代码后继续提问，并保留父问题、父回答、选中原文和新问题，使 AI 能准确理解用户所说的“这里”是什么。

### 4. 解决长对话中知识结构不清晰的问题

问题树会显示主问题、子问题、当前分支以及结论之间的来源关系，让用户看到的不只是聊天时间线，而是知识探索的逻辑结构。

### 5. 解决 AI 对话难以长期沉淀的问题

TreeTalk 可以把有价值的会话保存到本地知识仓库，包括问题节点、父子关系、节点回答、附件和完整的探索路径。

### 6. 解决不同 AI 接入方式的选择问题

TreeTalk 支持两种使用思路：

- **API 模式**：适合拥有模型 API Key、希望自行选择模型服务的用户；
- **浏览器桥接模式**：适合希望使用 ChatGPT 网页账户、不想单独配置 API 的用户。

## 主要功能

### 问题树与层级追问

- 围绕一个主问题创建多个子问题；
- 自动记录父问题和子问题关系；
- 支持连续深入多层问题；
- 支持返回父问题或跳转到任意节点；
- 避免多个研究方向互相干扰。

### 框选追问

- 支持框选普通文字；
- 支持框选代码片段；
- 支持针对数学公式继续提问；
- 自动携带相关上下文；
- 减少复制、粘贴和重新解释背景的操作。

### 逻辑树与问题图谱

- 可视化显示问题之间的逻辑关系；
- 查看完整问题分支；
- 快速识别当前问题的位置；
- 适合梳理课程知识、代码逻辑、数学推导和复杂概念。

### 流式回答与并行任务

- 回答逐步生成并显示状态；
- 阅读长回答时不会强制滚动到底部；
- 切换问题节点时，后台任务可以继续处理；
- 不同问题分支的生成结果不会互相覆盖。

### 代码、公式、图片与附件

- 支持 Markdown、代码块和行内代码；
- 使用 KaTeX 渲染数学公式；
- 支持从输入区添加附件和粘贴图片；
- 可用于分析截图、课程资料、代码图片和常见文档；
- 附件内容可以继续作为问题树的一部分进行追问。

### 本地知识仓库

- 将工作台中的问题树保存为知识文件；
- 使用文件夹组织不同主题；
- 支持文件和文件夹重命名、删除；
- 打开知识文件后可以继续追问；
- 临时工作台和长期知识仓库相互独立。

## 适合哪些人

TreeTalk 比较适合：

- 使用 AI 学习编程、数学、数据结构和网络技术的学生；
- 需要阅读论文、课程资料或技术文档的用户；
- 分析代码、报错和复杂逻辑的开发者；
- 希望保留完整排查过程的技术人员；
- 想利用 AI 获取知识，但希望将正式笔记交给其他笔记软件的用户；
- 认为普通 AI 聊天记录太长、分支关系不清晰的重度用户。

TreeTalk 不以多人协作、在线文档编辑或完整笔记数据库为主要目标。它主要解决的是：**如何更有结构地向 AI 提问、理解回答和沉淀问题路径。**

## 数据与隐私

TreeTalk 采用本地优先设计：

- 学习会话、知识仓库和附件默认保存在用户设备中；
- API Key、登录凭据和个人知识内容不应提交到 GitHub Issues；
- 使用第三方模型 API 或 ChatGPT 网页时，发送内容仍受对应服务的隐私政策约束；
- 公开反馈问题前，请移除日志或截图中的账号信息、私人文档和敏感路径。
