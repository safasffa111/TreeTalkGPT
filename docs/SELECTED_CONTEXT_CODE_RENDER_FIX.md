# 框选原文代码渲染修复

## 问题

框选原文卡片此前主要针对普通文本和公式做展示优化。用户从回答中的代码块框选内容时，浏览器返回的选区文本会丢失代码块语义，展示层会把它当成普通段落渲染，导致：

- 换行和缩进不稳定；
- 多行代码被普通段落折叠；
- 行内代码和代码块没有明显区分；
- 混合选择“文本 + 代码 + 文本”时，代码不能按代码块显示。

## 修复策略

### 1. 选区提取阶段识别代码节点

`selection-fingerprint.js` 新增对以下节点的识别：

- `.rich-code-block code`
- `.rich-inline-code`

当选区命中代码块时，按代码节点内部文本重新截取，保留原始换行和缩进，并包装为 Markdown 代码块：

```markdown
```js
const value = 1;
console.log(value);
```
```

当选区命中行内代码时，包装为 Markdown 行内代码：

```markdown
`npm.cmd start`
```

这样保存到 `selectedTextContext` 的仍是纯文本上下文，且更适合传给 AI 理解。

### 2. 展示阶段自动识别代码文本

`workbench-renderer.js` 的 `buildSelectedContextDisplayText()` 新增代码启发式识别：

- 如果已有 Markdown 代码围栏，直接交给 `rich-renderer`；
- 如果是多行且包含明显代码特征，临时包装成代码块；
- 如果是单行高置信代码，临时包装成行内代码；
- 公式识别仍然保留，但不会处理代码围栏内部内容。

### 3. 样式优化

`styles.css` 为 `.selected-context-body` 内的代码块增加轻量样式：

- 保留无边框设计；
- 代码区域使用轻微透明背景；
- 保留缩进、换行、横向滚动；
- 行内代码使用轻量背景。

## 注意

这次修改只改善框选原文卡片展示和代码上下文保真，不改变工作台问题树、历史列表、动画队列和持久化结构。
