# 知识仓库 MVP 用户流

基线版本：`v9_attach_menu_plus_button_anchor_final_fix`。

本轮实现的是用户指定的知识仓库第一阶段，而不是自动总结型知识仓库。

## 已实现

- 侧边栏新增“知识仓库”按钮。
- 原尺寸辅助栏显示知识仓库文件列表。
- 原尺寸辅助栏标题右侧有无边框新建按钮。
- 点击新建按钮会在当前列表创建文件夹，并立即进入重命名状态。
- 重命名为空时，点击其他区域或回车会自动命名为 `新建文件夹1/2/3...`。
- 知识仓库支持文件夹进入与返回上级。
- 文件项点击后会加载到内容栏，内容栏复用工作台内容渲染能力，可框选、公式/代码渲染、追问、出栈。
- 点击文件后，原尺寸辅助栏标题右侧按钮变为 `→`。
- 1.5 倍辅助栏逻辑图标题左侧新增无边框按钮。未选中文件时是预留按钮，点击不发生动作。
- 选中文件后，原尺寸/1.5 倍辅助栏箭头都可切换到该文件对应的逻辑树/逻辑图。
- 进入逻辑树/逻辑图后，箭头变为 `←`，点击会回到知识仓库列表状态，内容栏不变。
- 在知识文件内容栏中，问题栈为空时，删除按钮/Enter 不会返回历史列表，而是退出当前文本内容栏，并把辅助栏恢复为知识仓库列表状态。

## 数据存储

知识仓库目前使用本地 `localStorage`：

```text
ai-learning-stack.knowledge-warehouse.v1
```

文件夹/文件结构保存在知识仓库存储中；点击文件后，该文件会拥有独立的学习栈快照。

## 调试入口

控制台：

```js
window.__KNOWLEDGE_WAREHOUSE__.debug()
window.__KNOWLEDGE_WAREHOUSE__.debugCreateFile('测试文件', '这里是测试正文')
```

因为本轮用户只要求“新建文件夹”，没有要求 UI 新建文件，`debugCreateFile` 只是开发调试入口，方便验证“点击文件 → 内容栏 → 追问 → 树/图切换”的流程。
