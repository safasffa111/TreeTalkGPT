# 知识仓库辅助栏切换重叠最终修复

## 问题

从知识仓库切换到工作台、设置、错误日志等模块时，辅助栏可能出现旧知识仓库列表和新模块辅助栏短暂重叠。

根因是辅助栏面板都使用 `visibility` 延迟隐藏。模块切换时，旧面板的 delayed visibility 还没结束，新模块面板已经开始进入动画，导致一段时间内两个面板同时可见。

## 修复

1. `renderer.js` 在模块切换开始和应用新模块前，统一清理辅助栏运行时动画 class。
2. 模块切换前强制触发布局刷新，确保旧状态先被浏览器提交。
3. `styles.css` 增加 cross-module isolation：非当前模块的辅助栏面板强制 `visibility: hidden`，不再等待 delayed visibility。
4. 修正 `data-module-transition="in"` 下知识仓库列表被无条件设为 opacity 1 的问题。

## 验证点

- 知识仓库 → 工作台：不会叠加知识列表和历史列表/逻辑树。
- 知识仓库 → 设置：不会叠加知识列表和设置辅助栏。
- 知识仓库 → 错误日志：不会叠加知识列表和错误日志辅助栏。
- 快速连续点击模块按钮时，只显示最后目标模块的辅助栏。
