# DOCX MathType 公式保留与缓存失效修复

## 问题

针对包含 MathType / Equation OLE 对象的 `.docx`，上一版已经能在正文位置插入公式对象标记，例如：

```text
设[MathType公式#001: ...]且[MathType公式#002: ...]则...
```

但主 prompt 的输出规范同时要求“不要输出自定义占位符”。这会让模型把 `[MathType公式#编号: ...]` 当成不应输出的占位符，从而在提取题目时改写成：

```text
设……且……则……。
```

此外，附件解析缓存原本只按文件内容 hash 命中。用户用同一个 `.docx` 重测时，可能仍复用旧的提取结果，导致新解析器没有真正生效。

## 修复

1. MathType/OLE 公式输出格式从方括号占位式改为中文可读注记：

```text
【公式001（MathType近似转写）：\vec{a}=(1,k,2)】
```

2. 在附件上下文里明确提示：

```text
这些公式注记不是占位符，而是从 Word/MathType 公式对象恢复出的公式内容。
禁止删成“……”“缺失”或空白。
```

3. 修改全局输出规范：允许并要求保留附件解析中的 MathType 公式近似转写。

4. 增加附件解析缓存版本：

```js
ATTACHMENT_EXTRACTION_CACHE_VERSION = '2026-06-21-docx-mathtype-formula-preserve-v2'
```

旧缓存会自动失效，同一文件重新上传也会重新解析。

## 真实文件验证

对 `复件 高等数学D2复习题详细解答.docx` 的开头提取结果现在为：

```text
１、设【公式001（MathType近似转写）：\vec{a}=(1,k,2)】，【公式002（MathType近似转写）：\vec{b}=(-2,1,3)】且【公式003（MathType近似转写）：\vec{a}\perp \vec{b}】则【公式004（MathType近似转写）：k=】 .
```

不再是：

```text
设……且……则……。
```

## 边界

这仍然不是完整 MathType Native → LaTeX 引擎。复杂分式、矩阵、上下标的结构可能只能恢复为近似文本。但这版保证：

- 不把公式对象静默删掉；
- 不把公式内容变成“……”；
- 不因旧缓存继续使用上一版错误结果；
- AI 上下文会明确要求保留公式近似转写。
