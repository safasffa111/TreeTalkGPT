# 附件菜单遮挡修复

## 问题

在历史列表 / 辅助栏占用较大宽度时，工作台底部的“添加照片和文件 / 富文本”附件菜单可能向左越过工作区边界，被辅助栏或工作区 overflow 裁切，表现为菜单左侧被遮挡。

## 修复

- 将 `.attach-menu` 从基于 CSS 公式的绝对定位改为 `position: fixed`。
- 打开菜单时由 `attachment-controller.js` 根据：
  - `promptAttach` 按钮位置；
  - `.workspace` 实际边界；
  - 当前视口宽高；
  动态计算菜单位置。
- 菜单左侧会被限制在当前工作区内，不再滑到辅助栏下面。
- 监听 `resize` 和滚动事件，菜单打开时会重新计算位置。

## 影响范围

只影响附件菜单定位，不改变：

- 添加附件功能；
- 富文本模式；
- 输入框布局；
- 历史列表和内容栏动画；
- 后台请求生命周期。
