# 附件菜单锚点修复：对齐对话框左上方

本次修复基于 `v9_attach_menu_visibility_fix`。

## 问题

上一版附件菜单已经可以显示，并且不会被左侧辅助栏遮挡；但菜单定位仍然以 `+` 按钮附近为基础，在宽窗口下会显得漂移到输入框中间区域。

用户期望是：附件菜单出现在整个对话框最左边的正上方。

## 修复

`attachment-controller.js` 中的定位逻辑改为：

1. 获取 `.prompt-box` 的实际视口坐标。
2. 菜单左边缘优先对齐 `.prompt-box.left`。
3. 菜单顶部优先位于 `.prompt-box.top - menuHeight - gap`。
4. 继续使用 workspace 边界 clamp，避免菜单跑进左侧历史/辅助栏或窗口外部。

这样附件菜单在视觉上会稳定位于输入对话框左边缘正上方，而不是跟随 `+` 按钮漂移。
