# 混合文本 + 公式框选修复

## 问题

当用户在工作台内容栏中框选“文本 + KaTeX 公式 + 文本”时，浏览器原生 `selection.toString()` 有时只返回公式片段，导致：

- 输入框只判断到公式；
- 框选原文卡片只展示公式；
- 传给 AI 的追问上下文丢失公式前后的普通文本。

## 根因

KaTeX 会把一个公式拆成大量视觉 `span`。用户视觉上框选的是一整段内容，但浏览器的 Selection API 可能把公式内部 span 当成主要选区文本，导致普通文本片段被忽略或公式只返回局部视觉字符。

## 修复

`selection-fingerprint.js` 新增了 DOM 顺序重建逻辑：

1. 从实际 `Range` 和内容栏 body 开始遍历 DOM；
2. 普通文本节点按选区边界截取；
3. 遇到 `[data-math-source]` 的 KaTeX 包装节点时，不读取内部视觉 span，而是使用完整 LaTeX 源；
4. 按 DOM 顺序合并为：文本 + 完整公式 + 文本；
5. 公式会以 `\(...\)` 或 `\[...\]` 形式保留，方便显示层继续用 KaTeX 渲染，也保证传给 AI 的是完整公式。

## 效果

例如框选：

```text
这里由公式 $a^2+b^2=c^2$ 可以看出关系
```

保存为追问上下文时会接近：

```text
这里由公式 \(a^2+b^2=c^2\) 可以看出关系
```

显示层会继续用 KaTeX 渲染公式；发送给 AI 的仍是完整 LaTeX 源文本。
